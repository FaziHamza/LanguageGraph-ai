## 3. JSON Schema & Structure

### **MANDATORY Rule Structure**

```json
{
    "id": "string (unique, kebab-case)",
        "name": "string (human readable description)",
            "enabled": boolean,
                "priority": number(optional, higher = executed first),
                    "tags": ["string"](optional, for categorization),
                        "conditions": {
        // EITHER simple condition
        "field": "string (dot notation: emp.fieldName)",
            "operator": "string (exact from allowed list)",
                "value": any,
                    "config": { } (optional operator configuration),

        // OPTIONAL: Flow control
        "onSuccess": {
            "goto": "target-rule-id"
        },
        "onFailure": {
            "goto": "target-rule-id"
        },
        "onExit": {
            "goto": "target-rule-id"
        }

        // OR composite condition
        "logic": "all | any | not",
            "rules": [
                // Array of condition objects
            ]
    },
    "actions": [
        {
            "type": "string (exact from allowed list)",
            "target": "string (field/element identifier)",
            "value": any(action - specific value, optional for some actions),
        "options": {} // MANDATORY - always include even if empty
    }
  ]
}
```

### **Critical Requirements:**
- ‚úÖ ALWAYS use double quotes for strings
- ‚úÖ ALWAYS include "options": { }  in every action
- ‚úÖ NEVER use trailing commas
- ‚úÖ Boolean values must be lowercase (true/false not "true"/"false")
- ‚úÖ Numbers don't need quotes
- ‚úÖ Use exact operator and action names from allowed lists
- ‚úÖ Field references use dot notation (object.property)
- ‚úÖ Config options are optional but when used must be objects
- ‚úÖ Value field is optional for some actions (show, hide, enable, disable)

## 4. Field Reference Syntax

### **Field Naming Conventions**

| Context | Syntax | Example | Description |
|---------|--------|---------|-------------|
| Simple field | ```"fieldName"``` | ```"quantity"``` | Direct field reference |
| Object property | ```"object.property"``` | ```"emp.quantity"``` | Nested object field |
| Array element | ```"array[index]"``` | ```"items[0]"``` | Array index access |
| Deep nesting | ```"obj.sub.prop"``` | ```"user.profile.age"``` | Multi-level nesting |

### **Field Reference Syntax**

Use field names directly in calculate expressions:
```json
{
    "type": "calculate",
        "target": "total",
            "value": "{quantity} * {price} * (1 + {taxRate})",
                "options": { }
}
```

**CRITICAL: For string values and messages, use {fieldName} syntax:**
```json
{
    "type": "setValue",
        "target": "fullName",
            "value": "{firstName} {lastName}",
                "options": { }
}

{
    "type": "showMessage",
        "target": "messageArea",
            "value": "",
                "options": {
        "content": "Welcome {firstName}! Your order total is ${total}",
            "type": "success"
    }
}
```

**NEVER use JavaScript operators like + in JSON values:**
- ‚ùå WRONG: ```"content": "Hello " + firstName + "!"```
- ‚úÖ CORRECT: ```"content": "Hello {firstName}!"```

| 'workingDays' | 'holidays' | array | [] | Array of holiday dates to exclude | '{ "holidays": ["2025-12-25", "2025-01-01"] } ' |

## 9. Common Mistakes & Corrections

### **CRITICAL FIXES - Never Make These Mistakes**

```json
// ‚ùå WRONG: Missing options field
{
    "type": "setValue",
        "target": "field",
            "value": 100
}
// ‚úÖ CORRECT: Always include options
{
    "type": "setValue",
        "target": "field",
            "value": 100,
                "options": { }
}

// ‚ùå WRONG: Using custom class names instead of Tailwind classes
{
    "type": "addClass",
        "target": "tax-group",
            "value": "digital-tax-mode",
                "options": { }
}
// ‚úÖ CORRECT: Use direct Tailwind classes
{
    "type": "addClass",
        "target": "tax-group",
            "value": "bg-red-500 text-white border-2 border-red-600",
                "options": { }
}

// ‚ùå WRONG: String concatenation with + operator (breaks JSON)
{
    "type": "showMessage",
        "target": "pricing-section",
            "value": "",
                "options": {
        "content": "Profit Margin: " + "profitMargin" + "%",
            "type": "info"
    }
}
// ‚úÖ CORRECT: Use field reference syntax {fieldName}
{
    "type": "showMessage",
        "target": "pricing-section",
            "value": "",
                "options": {
        "content": "Profit Margin: {profitMargin}%",
            "type": "info"
    }
}

// ‚ùå WRONG: Missing config object structure
{
    "field": "price",
        "operator": "between",
            "value": [100, 500],
                "config": "inclusive"
}
// ‚úÖ CORRECT: Config must be an object
{
    "field": "price",
        "operator": "between",
            "value": [100, 500],
                "config": {
        "inclusive": true
    }
}

// ‚ùå WRONG: Incorrect value type for action
{
    "type": "show",
        "target": "section",
            "value": "visible",
                "options": { }
}
// ‚úÖ CORRECT: Show action doesn't need value
{
    "type": "show",
        "target": "section",
            "options": { }
}

// ‚ùå WRONG: Using non-existent operator
{
    "operator": "equals"
}
// ‚úÖ CORRECT: Use exact operator names
{
    "operator": "=="
}

// ‚ùå WRONG: Boolean as string
{
    "enabled": "true",
        "value": "false"
}
// ‚úÖ CORRECT: Boolean without quotes
{
    "enabled": true,
        "value": false
}
```
